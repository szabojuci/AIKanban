# DATABASE CONNECTION GUIDE

TAIPO now features a **fully driver-agnostic database layer**. You do not need to edit any PHP code to switch your database system! Everything is controlled purely through your `.env` configuration.

## 1. Configure your `.env` file

Open `backend/.env` (or duplicate it from `.env.example`) and locate the **Database settings** section:

```env
# -----------------------------
# SQLite Configuration
# -----------------------------
# If SQLITE_FILE_NAME is provided, the application will prioritize SQLite over server databases.
# To use MySQL/Postgres/etc, you MUST set this to "" or "None".
SQLITE_FILE_NAME="kanban.sqlite"
```

If **SQLITE_FILE_NAME** is set to **"None"** or **""**, the application will use the server database: by default **MySQL**/**MariaDB** is built-in the code. See `public static function getDatabaseConfig(): array` in [backend/src/Config.php:L72-L84](backend/src/Config.php)

```env
# -----------------------------
# Server Database Configuration
# -----------------------------
DB_TYPE=sqlite      # Options: sqlite, mysql, mariadb, pgsql, sqlsrv, oci
DB_HOST=localhost
DB_NAME=taipo_db
DB_USER=root
DB_PASS=secret
DB_PORT=3306        # MySQL/MariaDB: 3306 | Postgres: 5432 | SQL Server: 1433 | Oracle: 1521

# -----------------------------
# Table Prefixing
# -----------------------------
TABLE_PREFIX=""     # Examples: "taipo_", "dev_", or leave empty "" for no prefix
```

### ⚠️ Important Note on `TABLE_PREFIX`

The `TABLE_PREFIX` determines the exact names of the tables the application will read and write to (e.g., setting it to `taipo_` means the app uses `taipo_users` instead of `users`).

- **If you change this on an existing database**, the application will no longer find your old tables! It will assume it is a fresh install and will create a completely blank set of tables using the **new** prefix.
- If your application suddenly looks empty after an update, check to ensure your `TABLE_PREFIX` matches what it was originally!

## 2. Database Driver Magic (Now Automatic!)

> **Note:** The `Database.php` class has been heavily refactored so that you **DO NOT** need to manually perform SQL syntax adjustments when moving away from SQLite!

The application features **on-the-fly SQL normalization** and will automatically handle cross-database compatibility:

- **Connections:** Automatically parses the correct PDO Data Source Name (DSN) format for SQLite, MySQL, MariaDB, PostgreSQL, SQL Server, and Oracle.
- **Primary Keys:** Translates `INTEGER PRIMARY KEY AUTOINCREMENT` into `INT AUTO_INCREMENT PRIMARY KEY` (MySQL), `SERIAL` (PostgreSQL), `IDENTITY` (SQL Server), or `GENERATED BY DEFAULT AS IDENTITY` (Oracle).
- **Data Types**: Swaps `TEXT` for `CLOB` in Oracle or adapts `DATETIME` to `TIMESTAMP` in Postgres.
- **Dynamic Table Prefixing**: Appends `TABLE_PREFIX` to dynamically adjust all queries and table creation logic.
- **Schema Inspections**: Runs driver-specific column checks natively depending on the configured `DB_TYPE` (`SHOW COLUMNS`, `information_schema.columns`, `USER_TAB_COLUMNS`, or `PRAGMA table_info`).

> **NOTE**_ MAKE SURE YOU HAVE THE APPROPRIATE DRIVER INSTALLED ON YOUR SYSTEM and you have rights to use the database.
